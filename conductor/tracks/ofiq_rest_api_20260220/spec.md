# Track Specification: ASP.NET Core REST API for OFIQ

## Overview
Этот трек направлен на создание REST API сервиса на базе ASP.NET Core для предоставления функционала оценки качества лиц (OFIQ) через HTTP-интерфейс. Сервис будет использовать существующий C# SDK (NativeInvoke) и нативный мост.

## Functional Requirements
Сервис должен реализовать три основных метода (endpoints), принимающих изображение в формате IFormFile:

1.  **Метод A (/api/quality/scalar):**
    - Вход: Изображение (файл).
    - Выход: JSON с итоговым баллом качества (scalar quality score).
2.  **Метод B (/api/quality/vector):**
    - Вход: Изображение (файл).
    - Выход: JSON с координатами лица (Bounding Box) и полным списком метрик качества (Raw и Scalar значения).
3.  **Метод C (/api/quality/preprocessing):**
    - Вход: Изображение (файл).
    - Выход: Расширенный JSON, включающий все данные Метода B + координаты лендмарков лица (98 точек).

## Non-Functional Requirements
- **Платформа:** Работа исключительно на Linux (x64).
- **Документация:** Наличие Swagger UI для тестирования методов.
- **Производительность:** Использование оптимизированного Zero-copy механизма передачи данных в нативный код.
- **Безопасность:** Валидация входных файлов (проверка расширений и форматов изображений).

## Acceptance Criteria
- Все 3 метода корректно обрабатывают изображения и возвращают валидный JSON.
- API успешно инициализирует нативную библиотеку OFIQ при старте.
- Обработка ошибок (например, лицо не найдено) возвращает понятные HTTP статусы (400 Bad Request, 500 Error).
- Swagger доступен по адресу /swagger.
